# **프로그래밍 언어**
프로그래밍 언어는 컴퓨터 프로그램을 작성하는데 사용되는 하나의 표기 체계이다.
대부분 프로그래밍 언어는 문자들로 이루어진 형식언어이지만, 스크레치와 같은 gui 형식의 언어도 존재한다. [1]

## **기계어**
기계어는 기계(CPU)에서 바로 해석될 수 있는 0과 1로 된 인스트럭션들의 집합이다. 모든 프로세서는 각자의 인스트럭션 집합을 가지고 있고, 각 인스트럭션은 프로세서가 엄격히 정해진 작업을 수행하게 한다.
기계어는 그 인스트럭션이 기계(아키텍쳐)에 따라 다를 수 있으므로 (대부분의)고급 언어와 달리 기계 의존적이다. [2][3]

### **이진수**
이진수는 두 가지의 수(일반적으로 0과 1)로 이루어진 수 체계이다.
이진수를 통해 논리 게이트를 이용한 디지털 회로의 직접적 구현이 가능하기 때문에, 거의 모든 컴퓨터 장치들에서 사용된다. [4]

## **어셈블리어**
어셈블리어는 0과 1로 이루어진 기계어의 인스트럭션들을 사람이 알아보기 쉬운 문자 기호에 일대일 대응시킨 저급 프로그래밍 언어이다. [5]
기계어가 기계 의존적이므로 그에 일대일 대응하는 어셈블리어 또한 기계 의존적이다.

### **어셈블러**
어셈블러는 기계어 인스트럭션과 일대일 대응인 어셈블리어를 기계어로 치환시켜주는 프로그램이다. 반대로 리버싱에 사용되는 디스어셈블러도 있으며, 기계어를 어셈블리어로 변환해준다.

## **고급 프로그래밍 언어**
고급 프로그래밍 언어는 레지스터 관리, 메모리 주소 등의 컴퓨터의 여러 디테일을 추상화 시켜 인간이 이해하기 쉬운 형식으로 작성된 프로그래밍 언어이다.
고급 프로그래밍 언어는 일반적인 작업을 일반화 시키는 추상화의 특성상 아키텍쳐에 맞게 최적화 할 수 있는 저급 언어에 비해 효율성 면에서 떨어진다.
추상화의 정도가 프로그래밍 언어의 고급 정도를 결정 지으며, 대부분의 프로그래밍 언어는 고급 언어에 속한다. [6]

### **절차적 프로그래밍 언어**
절차적 프로그래밍 언어는 프로그래밍 패러다임의 하나로, 프로시저 호출에 기반한다. 프로시저는 함수라고도 불리며, 특정한 기능을 실행하기 위해 수행되어야 할 연산 절차들의 나열이다.[7][8]


#### **C**
C 언어는 1972년에 벨 연구소의 데니스 리치가 만든 다목적 절차지향적 프로그래밍 언어이며, 현대의 거의 모든 컴퓨터 아키텍쳐에 대응하는 C 컴파일러가 존재하므로 마이크로 컨트롤러부터 슈퍼컴퓨터에 이르기까지의 다양한 아키텍쳐에서 모두 사용된다. [1]

#### **포트란/코볼/파스칼**
포트란과 코볼은 50-60년대에 만들어진 초기 고급 프로그래밍 언어이고, 파스칼은 70년대에 프로그래머를 교육하기 위해 만들어진 언어이다.
포트란은 컴퓨터상에서 복잡한 수학식의 계산을 쉽게 하기 위해 만들어 졌으며, 코볼은 사무 자료 처리를 위해 만들어졌다.

#### **절차**
*아래 4가지는 모두 유사한 의미를 가져 '함수' 등으로 합쳐져서 불린다.*[8]
- 함수
> 함수는 특정한 기능을 실행하기 위해 수행되어야 할 순차적인 인스트럭션들의 나열이고, 호출이 가능하다.
> 특히 값을 입력받아 데이터를 처리하고 값을 리턴하기도 하는 코드를 지칭하는데 사용하기도 한다.
- 서브루틴
> 함수와 프로시저 모두를 지칭하는 말이며, 함수와 거의 유사하게 사용되는 단어이다.
- 프로시저
> 기본적으로 함수와 유사하지만, 특히 아무것도 리턴하지 않는 함수를 말한다.
- 메소드
> 메소드는 기본적으로 함수와 유사한 단어지만, 특히 객체지향 프로그래밍에서 한 객체속에 포함되어 있다는 것을 나타내며, 메서드를 호출하는 것을 통해 객체에 명령을 내린다.[9][10]

### **객체-지향 프로그래밍 언어**
객체지향 프로그래밍 언어는 '객체' 개념에 기반한 프로그래밍 패러다임의 하나이다.
기존 절차 지향 프로그래밍에서의 컴퓨터 프로그램을 명령어의 목록으로 보는 시각에서 벗어나, 여러 독립된 단위의 모임으로 파악하고자 하는 것이다.[10]
#### **객체**
객체란 클래스에서 정의한 것을 토대로 메모리에 할당된 공간으로, 변수, 자료구조, 함수(메소드)가 될 수 있으며, 한 객체는 하나의 자료 또는 명령을 포함하는 절차지향과는 달리 이들을 동시에 가질 수 있다.[10]
- 클래스
> 클래스는 어떠한 객체를 생성 하기 위해 변수와 함수들을 정의해놓은 일종의 틀이다. 클래스 끼리의 상속 등을 통해 다른 클래스의 일부 또는 전부를 어떠한 클래스에 포함시킬 수 있다.[11]
- 인스턴스
> 클래스를 실재적으로 메모리에 할당해 구현한 것으로, 클래스가 설계도라면 인스턴스는 실제 물건이라고 생각할 수 있다.[11]
#### **추상화**
추상화는 내부의 세세한 디테일을 숨겨, 외부에서 접근을 쉽게 만들어 준다.
추상화는 주로 데이터 추상화와 절차 추상화로 구분되며, 데이터 추상화는 외부에서 객체의 데이터를 직접 접근할 수는 없지만 메서드를 통해 접근할 수 있도록 하는 것이며, 절차 추상화는 객체 내부 함수의 구현적 디테일을 숨기고 객체 외부에서 메서드 호출을 통해 필요한 기능을 제공한다. [12][13]
#### **캡슐화(은닉화)**
캡슐화는 데이터와 해당 데이터를 처리하는 함수를 함께 묶은 것을 뜻한다. 이는 또한 객체의 데이터 또는 구성 요소의 직접적 접근을 제한한다는 의미도 된다.[14]
#### **상속**
상속은 객체들 간의 관계를 구축하는 방법 중 하나이며, 클래스는 상속을 통해 어떤 클래스로 부터 데이터와 함수 등을 받아올 수 있다. 상속 관계에는 상속 받는 입장인 서브클래스와, 상속하는 입장인 수퍼클래스가 있다.[15]
#### **다형성**
다형성이란 하나의 객체가 여러 타입들(상수, 변수, 함수, 오브젝트, 등)을 포함할 수 있음을 의미한다.[16]
단형성은 객체 지향 프로그래밍의 중요한 요소이다.
#### **JAVA**
자바란 1995년에 Sun Microsystems에 의해 만들어진 프로그래밍 언어와 컴퓨팅 환경을 지칭한다.
자는 고급 프로그래밍 언어이며 객체 지향 언어이다. 시스템 종속성을 줄이기 위해 가상 환경에서 실행되며, 자바 컴파일러는 자바 프로그램을 바이트코드라는 특수한 바이너리 형태로 변환한다.
실제로 시스템과 상관없이 jvm만 있다면 어느 시스템에서든 동일하게 실행된다.[17][18]
#### **C#**
C#은 마이크로소프트에서 개발한 객체 지향 프로그래밍 언어이다.
자바에서 jvm과 유사하게 CLR(common language runtime)이란 .NET(dotnet)언어들의 공통된 체제 위에서 실행된다.[19][20]

### **비절차적 언어**
비절차적 언어란 기계 상태에 따른 전형적인 프로그래밍 논리를 요구하지 않는 프로그래밍 언어이다.
사용자는 함수의 입/출력에만 집중하고, 프로그램의 절차에는 큰 신경을 기울이지 않아도 된다.[21]

### **마크업 언어**
마크업 언어는 태그 등을 이용하여 문서나 데이터를 구성하는 텍스트 인코딩 체계이다.[https://en.wikipedia.org/wiki/Markup_language]
일반적으로 문서나 데이터를 기술하는 용도로 사용되기에 프로그래밍 언어와는 구별된다.[https://ko.wikipedia.org/wiki/%EB%A7%88%ED%81%AC%EC%97%85_%EC%96%B8%EC%96%B4]

#### **HTML**
HTML은 Hyter Text Markup Language의 줄임말로, 웹 브라우져에서 보이도록 설계된 문서의 표준 규격이다.
HTML 문서는 여러개의 HTML 요소로 이루어져 있으며, 태그를 사용해 특정 요소임을 나타내어 이미지 표시, 하이퍼링크 등의 기능을 제공하는 구조화된 문서를 만들 수 있다.
HTML은 보통 CSS, JavaScript등과 함께 사용되며, 웹 브라우져는 CSS, JS 와 함께 구성된 HTML 문서를 그대로 표시하는 것이 아닌, 태그와 스크립트들을 일정한 규약에 맞게 랜더링하여 유저에게 표시해주는 역할을 한다. [https://en.wikipedia.org/wiki/HTML]

##### **태그**
태그는 마크업 언어에서 사용되는 일종의 지시어이다.
태그는 마크업 언어로 작성된 문서에서 텍스트 영역 사이에 넣어지며, 해당 태그는 의도된 목적, 해당 텍스트의 의미 등을 나타낸다.
예를 들어 HTML의 `<h1>` 태그의 경우 해당 태그 사이의 텍스트 영역이 1단계 헤딩(≈제목)임을 의미하고, 랜더링시 텍스트의 크기 등을 정해진 규격에 맞게 변환하여 표시해주도록 지시하는 역할을 한다. [https://en.wikipedia.org/wiki/Markup_language]
