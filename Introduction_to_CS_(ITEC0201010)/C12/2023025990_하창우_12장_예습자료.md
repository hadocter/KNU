# 빅데이터
빅데이터는 일차적으로 일반 데이터 처리 소프트웨어를 통해 처리하기 어려울 정도로 크거나 복잡한 데이터를 뜻하며, 적은 양의 정보로는 인사이트를 생성하기 어렵다. [2]

## 종류

### 내부 데이터
내부 데이터란 시스템(조직) 내부에서 수집한 데이터로, 대부분 형식과 규칙이 정해져 있는 정형 데이터이다. [1][4]

### 정형 데이터
정형 데이터란 표준화된 형식을 따르고 구조가 잘 정의되어 있는 데이터를 의미한다. 정형 데이터는 보통 관계형 데이터베이스에서 저장, 관리 된다. [3]

### 반정형 데이터
반정형 데이터란 정형 데이터만큼 구조에 대해 엄격하지는 않지만, 일부 구조적인 요소가 있는 데이터를 의미한다.
태그 혹은 메타데이터를 통해 약간의 구조를 설명하지만, 보통 관계형 데이터베이스에 저장할 수 없다. [3]

### 비정형 데이터
비정형 데이터는 어떠한 모델을 따르지 않으며, 구조를 쉽게 식별할 수 없는 데이터를 의미한다.
구조가 없으므로 저장 밎 관리 활용에 어려움이 따른다. [3]

### 외부 데이터
외부 데이터란 인터넷 등을 통해 시스템 외부에서 수집하는 데이터로, 협약 등의 별도 방법을 취하지 않는 이상 대부분 비정형 데이터이며, 데이터 수집이 불가능한 경우도 존재한다. [4]

## 빅데이터 컴퓨팅
빅데이터는 기존의 데이터 처리 방식을 통해 처리하기 어려우므로, 빅데이터를 효율적으로 처리할 새로운 방법들이 필요하다. 이 방법들을 통틀어 빅데이터 컴퓨팅이라 한다.
### 빅데이터 수집
빅데이터의 수집은 주로 프로그램 등을 통해 자동으로 이루어진다. 서버 등의 로그 수집, 웹 크롤링, 센서의 데이터 받아오기, ETL 등의 방법이 있다.
수집된 데이터는 보통 데이터 웨어하우스, 분산 처리 시스템에 저장된다.[3]
#### 배치 처리
하둡 등에서 사용되는 방식으로, 여러 서버에 데이터를 분산하여 저장했다가, 데이터를 처리할 때 처리 코드를 서버로 전송하여 각각의 서버에서 데이터를 처리하고 처리 결과를 다시 모아 정리하는 방식을 의미한다. [5]
#### 실시간 처리
데이터를 저장했다가 일정한 기준에 따라 데이터 처리 엔진에 입력되는 배치 처리와 달리, 실시간 처리는 데이터가 생성된 후 짧은 시간 이내에 데이터 처리 엔진에 입력된다. 이때 실시간 데이터 처리 엔진은 지속적으로 작동된다.[3]

### 빅테이터 처리/저장
다양한 형식의 대용량 데이터인 빅데이터를 효율적으로 저장하고 처리하는 과정이다. [1]
#### 빅데이터 전처리
전처리 과정에서는 필요없는 부분을 제거하는 필터링, 분석이 쉬운 유형으로 데이터를 변환하는 과정, 데이터의 이상치, 손실값, 중복값 등을 제거하여 데이터의 신뢰성을 높이는 과정이 수반된다. [1]
#### 빅데이터 후처리
후처리 과정에서는 다양한 형식의 데이터를 일정한 형식으로 변환하여 분석이 용이하도록 한다.
이 과정에서 평활화, 집계, 일반화, 정규화, 속성 생성의 단계를 거친다.[1]
#### 데이터 통합
출처가 다른 데이터를 관련성에 따라 결합하는 것을 의미한다. 데이터 통합을 통해 여러 종류의 데이터를 이용한 분석을 수행할 수 있다. [7]

### 빅데이터 분석
빅데이터 분석이란, 다양한 형태의 데이터가 존재하며 빠르게 변화하는 방대한 데이터 세트에서 인사이트를 만들어 내는 것이다.[8]
#### 데이터 마이닝
데이터 마이닝이란 대규모의 데이터 안에서 체계적인 방식을 이용해 데이터의 통계적인 규칙을 발견하여 가치있는 정보를 추출하는 과정이다.[9]
#### 통계분석
합리적 의사결정을 위해 통계적인 방법에 입각하여 분석하는 행위를 말한다.
일변량분석, 다변량분석등이 있다. [14]

#### 분류
분류란 데이터를 미리 정해놓은 어떠한 기준에 따라 분류하는 것을 의미한다.
#### 군집
주어진 데이터 집합 내부에서 유사한 데이터들의 그룹으로 나누는 것을 의미한다.
비지도학습의 한 분야이고, 분류와는 다르게 인간이 미리 정한 기준이 없다. 대표적인 방법으로 k-mean 방법이 있다. [10]
#### 상관관계 분석
상관관계 분석이란 두 변수간의 선형적 관계를 계수로 수치화 시키는 것을 뜻한다.
두 변수의 인과관계는 고려하지 않고, 관계의 정도만을 조사한 [11]
#### 스트림 데이터 분석
스트림 데이터 분석은 데이터를 일괄(배치)처리하지 않고 연속적으로 처리 및 분석하는 것을 뜻한다.
시간에 민감한 데이터를 분석하는데 사용된다. [12]
#### 시각화
데이터 시각화란, 수집된 데이터의 분석 결과를 쉽게 알 수 있도록 시각적으로 표현하는 기술을 말한다.
차트, 표, 그래프 등의 방식이 존재한다. [12]

## 특징(5V)
### 규모(Volume)
생성, 혹은 수집된 데이터의 양을 의미하고, 해당 데이터의 빅데이터인지의 여부는 데이터의 규모에 따라 결정된다. 또 데이터의 가치와 데이터가 가질수 있는 인사이트는 해당 데이터의 규모에 의해 결정된다. 빅데이터의 규모는 보통 수 테라바이트~페타바이트 이상이다. [2]

### 속도(Velocity)
요구사항을 충족하기 위해 필요한, 데이터의 생성 및 처리 속도를 의미한다. 
예를 들면 환자를 모니터하기 위한 웨어러블 또는 병원의 장비들에게서 수집한 데이터는 빠르게, 실시간 수준으로 분석되어야 한다.


### 다양성(Variety)
빅데이터에서 처리하는 데이터는 정형, 반정형, 비정형등으로 다양한 종류의 데이터를 수용함을 의미한다.[5]

### 진실성,정확성(Veracity)
데이터에 부여할 수 있는 신뢰 수준을 의미한다.
방대한 데이터를 분석하여 패턴을 추출했을 때, 데이터가 많아질수록 엉터리 데이터 또한 많아질 수 있기 때문에 수집한 데이터가 정확한 것인지, 분석할만한 가치가 있는지 등을 살펴야 하는 필요가 생겼고, 이러한 측면에서 데이터에 부여할 수 있는 신뢰 수준을 의미하는 정확성이 빅데이터의 특징으로 제시되었다.

### 가치(Value)
빅데이터는 기업 등의 현실 문제 해결에 도움을 주는 통찰을 가진다.
